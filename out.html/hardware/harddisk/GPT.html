GPT - GUID Partition Table

GPT fixes some/many problems of MBR - Master Boot Record.
GPT needs UEFI - Unified Extensible Firmware Interface.

Layout on harddisk

A GPT disk has still an MBR - Master Boot Record partition table on the first sector of the disk (the protective MBR) to prevent MBR-only partitioniong tools from overwriting GPT disks.
When executing gdisk /dev/sda, it might print (note the Found valid GPT with protective MBR): 
GPT fdisk (gdisk) version 1.0.0

Partition table scan:
  MBR: protective
  BSD: not present
  APM: not present
  GPT: present

Found valid GPT with protective MBR; using GPT.

Command (? for help):
The GPT is (usually?) in the 2nd sector (LBA 1).
The first partition (usually) starts with sector 2048.

Partition type

With parted: parted /dev/disk set partition-number bios_grub on
With gdisk: use partition type 0xEF02.
Otherwise, set GUID to 21686148-6449-6e6f-744e656564454649.
