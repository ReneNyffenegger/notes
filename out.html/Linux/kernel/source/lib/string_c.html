lib/string.c

Stupid (string related) routines, obviously buggy: strncasecmp, strcasecmp, strcpy etc
Functions only compiled if __HAVE_ARCH_STRNCASECMP, __HAVE_ARCH_STRCASECMP, __HAVE_ARCH_STRCPY etc not defined.
Optimized (and less buggy?) arch dependent versions for example in arch/x86/include/asm/string_64.h.
